Projects

Here are some personal projects that I have been working on. Most of them are still a work in progress, will all the code found on my github.

Text-based Role-playing Fighter

This one is a turned-based text role-playing text fighter, inspired by mechanics from Pokemon and World of Warcraft. Players can play as one of multiple classes, and take on various monsters. Each class has a different playstyle, like the melee-focused warrior, the spell-casting mage, disruption-based priest, or the versatile shapeshifter. Other mechanics include items like potions, armor and weapons to help add some more variations. The program outputs to the console, but I have intentions to create a better interface, as well as adding exploration mechanics to make this game a more full-fledged rpg.

Checkers Tournament

This project was initially made to be used as a lab exercise to students in order to apply game theory concepts in a programming context. Each student implements a checkers board analyzer, which is then used to run students in a round-robin style checkers tournament. Each checkers match takes two players, where each move is determined by using a greedy search to predict the upcoming turns based on the students' board analyzers. All of the output is currently in a console representation. I plan to change it to a more interactive GUI in the future.

Music Identifier

Based on an implementation I found online, this program mimics the baseline functionality of the Shazam music discovery app. The program listens to an audio sample, breaks it down into key frequency points using a Discrete Fourier transformation, and matches those points against preexisting values. To work properly, songs need to be parsed and stored before they can be identified. The main issue is that I still have to find a way to store the parsed values in a fast and space efficient manner. The initial parse method can also be improved, as only local audio files currently work.

Game of Life

Look at that, another game. This one is more in the sense that it is a cellular simulation, and less of an interactive game. The behavior implements the rules of Conway's Game of Life in Python, along with having some simple starting patterns like the glider, glider gun, and toad. Currently the application outputs to either the console or the curses terminal, but I plan to add an actual GUI interface and better user input to allow for custom patterns.
